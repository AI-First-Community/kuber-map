# Getting help

Thanks for using **Kuber Map**! Here's the fastest way to get what you need.

## I have a question / want to discuss an idea
Use **[Discussions](https://github.com/AI-First-Community/kuber-map/discussions)** — for "how does
this work?", "would a use case for X make sense?", or sharing what you built. It keeps open-ended
conversation out of the issue tracker.

## I want to read the docs
The **[wiki](https://github.com/AI-First-Community/kuber-map/wiki)** is the home for detail:
- [Getting Started](https://github.com/AI-First-Community/kuber-map/wiki/Getting-Started) — set up and run it locally
- [Vision & Philosophy](https://github.com/AI-First-Community/kuber-map/wiki/Vision) — why this exists
- [For AI Teams](https://github.com/AI-First-Community/kuber-map/wiki/For-AI-Teams) — using the context packs to ground an agent
- [Authoring a Use Case](https://github.com/AI-First-Community/kuber-map/wiki/Use-Case-Authoring) — the most common contribution

## I found a bug or want to propose something
Open an **[issue](https://github.com/AI-First-Community/kuber-map/issues/new/choose)** with the right
template:
- **Bug report** — something is broken.
- **Use-case proposal** — a new curated slice of FIBO.
- **FIBO fidelity** — the map misrepresents FIBO (cite the source IRI; these take priority).

Looking for a first contribution? See the
**[good first issues](https://github.com/AI-First-Community/kuber-map/contribute)**.

## I found a security problem
Please **don't** open a public issue — follow **[SECURITY.md](SECURITY.md)** (private reporting).

## Before you post
- Search existing issues and discussions first — someone may have already asked.
- For contributions, read **[CONTRIBUTING.md](CONTRIBUTING.md)** and the
  **[Code of Conduct](CODE_OF_CONDUCT.md)**.

This is a volunteer-run community project, so replies are best-effort — thanks for your patience.
